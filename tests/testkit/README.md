# Testkit Scaffold

Hier kommen nur gemeinsame Helfer hinein, keine Produktivlogik.

Noch selbst anzulegen:
- builders/ fuer Order-, Basket-, Product- und Snapshot-Builder
- fixtures/ fuer wiederverwendbare Seed- und Szenariodaten
- fakes/ fuer kontrollierte Provider-, Broker- oder Clock-Fakes
- http-clients/ fuer gemeinsame API-Aufrufe
- event-factories/ fuer kanonische Event-Payloads
- assertions/ fuer wiederkehrende Fachassertions
