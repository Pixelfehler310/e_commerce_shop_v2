Bestellprozess

Happy Path und Kompensation

Die Saga ist das Zentrum der Architektur. Jeder Schritt erzeugt Status, Logs und Audit-Snapshots.

  1. Order wird im Shop-Service angelegt
  2. Warehouse reserviert Bestand mit Datenbanksperre
  3. Billing führt Payment über Fassade aus
  4. Invoice erzeugt asynchron eine PDF-Rechnung
  5. Warehouse bucht final aus, Shop schließt Order ab

Kompensationen

  • Zahlung abgelehnt: Reservierung stornieren, Order `PAYMENT_FAILED`.
  • Lager nicht ausreichend: keine Zahlung starten, Order `OUT_OF_STOCK`.
  • Booking fehlgeschlagen: Refund auslösen, Order `ROLLBACK_COMPLETED`.
  • Invoice ausgefallen: kein Payment-Rollback, Retry und Circuit Breaker.

Präsentation

Demo-Szenarien

Die Szenarien sind so ausgelegt, dass Architekturentscheidungen live sichtbar werden.

Happy Path Bestellung, Zahlung, Rechnung, Ausbuchung. Payment abgelehnt Reservierung wird kompensiert. Booking-Fehler Refund nach erfolgreicher Zahlung. Concurrency Zwei parallele Käufe, kein Überverkauf.
